# MySQL数据库自动化巡检系统使用说明

## 一、系统简介

本系统采用 **Shell巡检 + Python分析** 双层架构，一站式完成MySQL数据库的自动化巡检和报告生成。

```
┌─────────────────────────────────────────────────┐
│  目标服务器（MySQL所在机器）                       │
│  mysql_inspection.sh  →  采集巡检数据 →  .log文件  │
└─────────────────────────────────────────────────┘
                         ↓
┌─────────────────────────────────────────────────┐
│  分析端（任意机器）                                 │
│  log_analyzer.py    →  分析单个日志  →  Word/报告   │
│  batch_analyze.py   →  批量分析日志  →  汇总报告    │
│                                       →  Excel统计表 │
└─────────────────────────────────────────────────┘
```

---

## 二、功能速览

| 功能 | 命令 | 产出 |
|------|------|------|
| 采集巡检数据 | `mysql_inspection.sh` | `.log` 巡检日志文件 |
| 分析单个实例 | `log_analyzer.py` | 单个实例的 Word/文本 报告 |
| 批量分析实例 | `batch_analyze.py` | 批量汇总 Word 报告 + 统计摘要 |
| 生成统计表格 | `batch_analyze.py --excel` | Excel 统计报表（含 P0/P1/P2 问题分布） |

## 三、快速开始

### 3.1 环境准备

**目标服务器**（MySQL 所在机器）：
- 需安装 MySQL 客户端
- 具备至少 `SELECT` 权限的数据库用户

**分析端**（运行 Python 脚本的机器）：
```bash
# 安装依赖（可选，不安装则只能生成 txt 文本报告）
pip install -r requirements.txt
```

依赖包说明：
| 包名 | 用途 | 缺省影响 |
|------|------|---------|
| `python-docx` | 生成 Word 报告 | 缺少则只能生成 txt 报告 |
| `openpyxl` | 生成 Excel 统计表 | 缺少则 `--excel` 功能不可用 |

### 3.2 完整流程示例

```bash
# 步骤1：上传巡检脚本 在目标服务器执行巡检 
chmod +x mysql_inspection.sh
./mysql_inspection.sh --host 192.168.1.100 --port 3306 --user root --password abc123

# 步骤2：将生成的 .log 文件传到分析端机器

# 步骤3：分析单个实例（生成 Word 报告）
python log_analyzer.py inspection_logs/mysql_inspection_192.168.1.100_3306_20260424.log

# 步骤4（可选）：批量分析所有实例（生成汇总 Word + Excel 统计表）
python batch_analyze.py inspection_logs/ --excel
```

## 四、Shell 巡检脚本（mysql_inspection.sh）

### 4.1 命令行参数

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `--host` | `-h` | MySQL 主机 IP | `localhost` |
| `--port` | `-P` | MySQL 端口 | `3306` |
| `--user` | `-u` | 连接用户名 | `root` |
| `--password` | `-p` | 连接密码 | 空 |
| `--socket` | `-S` | Socket 文件路径（优先级高于 host+port） | 空 |
| `--output-dir` | `-o` | 日志输出目录 | `./inspection_logs` |
| `--rds-check` | - | RDS 环境（跳过系统资源检查） | - |
| `--mgr-check` | - | 检查 MGR 集群 | - |
| `--pxc-check` | - | 检查 PXC 集群 | - |
| `--help` | - | 显示帮助信息 | - |

### 4.2 使用示例

```bash
# 基本用法（默认连接 localhost:3306）
./mysql_inspection.sh

# 指定 host+port
./mysql_inspection.sh -h 192.168.1.100 -P 3307

# 指定用户名密码
./mysql_inspection.sh -h 192.168.1.100 -u admin -p mypassword

# 使用 Socket 连接（优先级高于 host+port）
./mysql_inspection.sh -S /var/run/mysqld/mysqld.sock

# 检查 MGR 集群
./mysql_inspection.sh -h 192.168.1.100 -P 3306 --mgr-check

# 自定义输出目录
./mysql_inspection.sh -h 192.168.1.100 -o /data/logs

# 完整参数组合
./mysql_inspection.sh -h 192.168.1.100 -P 3306 -u admin -p mypassword -o ./logs --mgr-check
```

### 4.3 输出文件

日志文件保存在 `--output-dir` 目录（默认 `inspection_logs/`），命名格式：

```
mysql_inspection_{IP}_{PORT}_{时间}.log
mysql_inspection_socket_{时间}.log          # 使用 socket 连接时
```

## 五、日志分析器（log_analyzer.py）

### 5.1 命令行用法

```bash
python log_analyzer.py <日志文件路径> [输出格式]
```

| 参数 | 说明 |
|------|------|
| `日志文件路径` | 巡检生成的 `.log` 文件（必填） |
| `输出格式` | `docx`（默认，Word 报告）或 `txt`（文本报告） |

### 5.2 使用示例

```bash
# 生成 Word 报告
python log_analyzer.py inspection_logs/mysql_inspection_192.168.1.100_3306_20260424.log

# 生成文本报告
python log_analyzer.py inspection_logs/mysql_inspection_192.168.1.100_3306_20260424.log txt
```

### 5.3 报告内容

Word 报告包含以下章节：

| 章节 | 内容 |
|------|------|
| 封面 | 报告标题、实例 IP/端口、生成日期 |
| **目录** | **自动生成的目录，涵盖所有章节标题和页码，支持点击跳转（打开文档时自动更新）** |
| 一、执行摘要 | 问题统计：P0（严重）/ P1（警告）/ P2（提示）问题数量 |
| 二、巡检项详情 | 按类别展示所有巡检项的检测结果和健康状态 |
| 三、错误日志分析 | 分析 MySQL 错误日志，识别连接错误、死锁、内存等异常模式 |
| 四、问题详情 | 逐条列出所有发现的问题及优化建议 |
| 五、处理建议 | 按优先级汇总所有处理建议 |

> **注意**：对于 MGR 或 PXC 集群，系统会自动识别并跳过传统主从同步检查，在巡检项详情中显示"当前实例为MGR/PXC集群，无需传统主从同步检查"，避免误报"从库复制中断"等问题。

### 5.4 报告输出路径

报告保存在当前目录的 `reports/` 文件夹下，命名格式：

```
MySQL_inspection_{IP}_{PORT}_{日期}.docx
MySQL_inspection_{时间戳}.docx              # 无法获取 IP 时
```

## 六、批量分析脚本（batch_analyze.py）

### 6.1 命令行参数

```bash
python batch_analyze.py <日志目录> [选项]
```

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `日志目录` | - | 存放 `.log` 文件的目录（必填） | - |
| `--format` | `-f` | 输出格式：`docx` 或 `txt` | `docx` |
| `--workers` | `-w` | 并行分析进程数 | CPU 核心数-1 |
| `--excel` | - | **额外生成 Excel 统计表格**（需安装 openpyxl） | 不生成 |

### 6.2 使用示例

```bash
# 基本用法：批量分析目录下所有日志，生成 Word 汇总报告
python batch_analyze.py inspection_logs/

# 指定并行度
python batch_analyze.py inspection_logs/ -w 4

# 批量生成文本报告
python batch_analyze.py inspection_logs/ -f txt

# 批量分析 + 生成 Excel 统计表格
python batch_analyze.py inspection_logs/ --excel

# 组合使用
python batch_analyze.py inspection_logs/ --excel -f docx -w 4
```

### 6.3 输出文件

输出保存在 `batch_reports_YYYYMMDD_HHMMSS/` 目录下：

| 文件 | 说明 |
|------|------|
| `批量分析摘要.txt` | 分析结果摘要，列出成功/失败文件 |
| `MySQL批量巡检汇总报告.docx` | **批量汇总 Word 报告**，包含封面、自动目录、汇总执行摘要、各实例详细问题，目录支持点击跳转 |
| `巡检统计报表_YYYYMMDD_HHMMSS.xlsx` | **Excel 统计表格**（仅 `--excel` 时生成） |

### 6.4 Excel 统计表格说明

使用 `--excel` 参数时生成，包含两个 Sheet：

**Sheet 1 - 问题统计汇总：**

| 列 | 说明 |
|----|------|
| 序号 | 实例编号 |
| IP 地址 | 实例 IP（从日志文件名自动提取） |
| 端口 | 实例端口 |
| P0 严重问题 | 严重问题数量（红色标识） |
| P0 问题描述 | 列出前 8 个 P0 问题的分类和标题 |
| P1 警告问题 | 警告问题数量（橙色标识） |
| P1 问题描述 | 列出前 8 个 P1 问题的分类和标题 |
| P2 信息提示 | 提示数量（绿色标识） |

- 支持自动筛选、隔行变色
- 底部合计行汇总各等级总数

**Sheet 2 - 问题详情：**
逐条展示每个问题的：级别、分类、问题描述、优化建议，按实例和级别顺序排列。

## 八、定时自动巡检

### Linux Crontab 示例

```bash
# 每天凌晨 2 点执行巡检
0 2 * * * /path/to/mysql_inspection.sh -h 192.168.1.100 -P 3306 -u root -p mypassword -o /data/logs

# 每周一凌晨 3 点执行巡检（socket 方式）
0 3 * * 1 /path/to/mysql_inspection.sh -S /var/run/mysqld/mysqld.sock -o /data/logs
```

巡检完成后，将日志文件传输到分析端后执行批量分析：
```bash
python batch_analyze.py /data/logs/ --excel
```

## 九、常见问题

### 9.1 Shell 巡检问题

```bash
# MySQL 连接失败：检查服务状态
systemctl status mysql

# 检查网络连通性
ping 192.168.1.100

# 检查防火墙
iptables -L | grep 3306

# Socket 连接：确认 socket 文件存在
ls -l /var/run/mysqld/mysqld.sock

# 用户权限不足
mysql -u root -p -e "GRANT SELECT ON *.* TO 'monitor'@'%';"
```

### 9.2 Python 分析问题

```bash
# 缺少 python-docx：只能生成 txt 报告
pip install python-docx

# 缺少 openpyxl：无法使用 --excel 参数
pip install openpyxl

# 一键安装所有依赖
pip install -r requirements.txt
```

### 9.3 参数冲突说明

- 同时指定 `--socket` 和 `--host` 时，**socket 参数优先级更高**
- `-h` 作为简写有特殊处理：单独使用 `-h` 显示帮助，配合参数使用 `-h IP` 指定主机
